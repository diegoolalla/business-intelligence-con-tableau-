# business-intelligence-con-tableau-
Tarea business intelligence con tableau 


En este caso práctico de Easy Loans debes analizar las operaciones de préstamos realizadas durante 2023 en Tableau Desktop con el objetivo de extraer insights que ayuden a la toma de decisiones estratégicas, identificando patrones de comportamiento, evaluando la calidad de los préstamos y proponiendo recomendaciones para mejorar la eficiencia y la rentabilidad de las operaciones financieras, aplicando además técnicas avanzadas de visualización para detectar anomalías y oportunidades. Para ello, trabajarás con el archivo Easy Loans Operaciones 2023.xlsx, revisando desde el inicio la calidad de los datos importados (tipos de datos, valores nulos e inconsistencias) y construyendo un modelo de datos correcto mediante relaciones entre las tablas Orders (order_id, created_at, status, amount, merchant_id, country), Refunds (order_id, refunded_at, amount) y Merchant (merchant_id, name); después deberás filtrar el dataset para que solo se incluyan operaciones en tiendas europeas, excluyendo específicamente Marruecos, y generar una extracción (“Extraer”) guardando el trabajo en formato .twbx para que sea compartible sin problemas. A nivel de exploración y análisis, se solicita crear cálculos en Tableau que permitan medir el rendimiento del negocio: el Promedio del importe de préstamos para entender niveles típicos de financiación y sus variaciones; el Total Comercios para dimensionar la cobertura y distribución; el Máximo y Mínimo para localizar operaciones extremas y analizar posibles riesgos; el Valor Acumulado usando RUNNING SUM para ver la evolución temporal y detectar patrones de crecimiento; el Promedio Total mediante una LOD FIXED para fijar el promedio global y poder compararlo con segmentos; y el Total Reembolsos para cuantificar devoluciones y reflexionar sobre políticas que reduzcan reembolsos. Además, debes crear un parámetro interactivo llamado “Valor Préstamo Mínimo” (numérico, con valor inicial por ejemplo 0) y un campo calculado del tipo Amount > [Valor Préstamo Mínimo], que se usará como filtro en todas las visualizaciones seleccionando únicamente el valor True, mostrando el parámetro dentro del dashboard (no el filtro) para que el usuario pueda ajustar dinámicamente el umbral y ver cómo cambian todas las vistas en tiempo real. En cuanto al desarrollo del contenido, debes construir una tabla de KPIs que reúna “Valor Acumulado”, “Promedio”, “Promedio Total”, “Máximo”, “Mínimo”, “Total Comercios” y “Total Devoluciones”, asegurando cálculos correctos y formatos legibles (separadores de miles, etc.); crear un mapa por país coloreado por el “Promedio” con un gradiente que permita distinguir claramente valores altos y bajos; construir un gráfico de áreas que muestre la suma acumulada diaria (“Valor Acumulado”) por día, diferenciando por país mediante color; y crear una visualización de desviación para identificar operaciones por encima o por debajo del promedio, usando un cálculo como Promedio - AVG([Promedio Total]) y aplicando un color escalado de 2 pasos y ordenado para resaltar diferencias (por ejemplo, encima vs. debajo). También debes incorporar filtros globales de “País”, “Fecha de Creación” como rango y el filtro basado en el campo calculado del “Límite Préstamo Mínimo” (solo True), asegurando que aplican a todas las visualizaciones, y finalmente integrar todo en un dashboard final bien maquetado, con títulos y etiquetas que faciliten la interpretación incluso a usuarios no técnicos. Como parte de la interactividad, tienes que configurar una acción para que al hacer clic en un país del mapa se filtren automáticamente el resto de visualizaciones (activando el icono del embudo y verificando que el filtrado afecta a las demás hojas del dashboard). De forma opcional puedes mejorar la experiencia de usuario con un diseño alternativo más personal (estética, navegación, iconos, contenedores ocultables, selectores de métricas o visualizaciones extra), y como bonus puedes publicar el dashboard en Tableau Cloud compartiéndolo solo con el profesor (título con tu nombre, permisos correctos, captura y enlace) o en Tableau Public (también con captura y enlace). La entrega obligatoria se realiza por el Campus Virtual e incluye el archivo .twbx con todas las hojas, dashboards, filtros, cálculos, parámetro y acciones funcionando correctamente; si optas al bonus, también se entrega un PDF con la evidencia de publicación (captura de pantalla y enlace), y antes de entregar se exige revisar que todo carga bien, que no hay errores de conexión, que las interacciones funcionan y que el archivo se abre correctamente en otro equipo para garantizar que el profesor podrá evaluarlo sin incidencias.
